# Vault 테스트 시나리오 (보험설계사 기준)

> 목적: 핵심 기능을 순서대로 검증하고, VOC(피드백)를 받기 위한 대본.
> 환경: `C:\Vault\KnowledgeVaultWeb.exe` 더블클릭 → 브라우저 http://localhost:8765

---

## 0. 준비 (최초 1회)
exe는 빈 상태로 시작합니다. 대시보드·제안서 테스트를 하려면 샘플 데이터를 넣으세요.

1. `C:\Vault\` 에 `KnowledgeVaultWeb.exe` + `key.txt` (완료)
2. **샘플 데이터 복사** — 아래 폴더들을 `C:\Vault\` 안으로 복사:
   - `Documents\insurance\고객DB`
   - `Documents\insurance\상품DB`
   - `Documents\insurance\프롬프트.md`
3. exe 더블클릭 → 브라우저 자동 → 상단에 **Vault** 보이면 성공

---

## 시나리오 A — 흩어진 자료 정리 + 질문 ⭐(핵심)
**상황:** 고객 관련 엑셀/문서가 여기저기 흩어져 있다.

1. **💬 AI 질문** 탭 → **파일 선택** → 엑셀/PDF 파일 추가
2. "✅ 추가됨 (인덱스 N건)" 확인
3. 질문 입력: `방금 넣은 자료 요약해줘`
4. → AI가 자료 내용을 근거로 답하는지 확인

✅ **기대:** 어떤 파일이든 MD로 변환·인덱싱되고, AI가 그 내용으로 답한다.

---

## 시나리오 B — 고객 분석 + 제안서·문자 ⭐(영업)
1. **📊 대시보드** 탭 → 갱신 임박 고객 확인 (정수남 D-9 등)
2. **💬 AI 질문**: `정수남 고객 갱신 상황과 추천 액션 알려줘`
3. **⚙️ 설정** → "수동 프롬프트 만들기" → 고객 **정수남** 선택
4. **② 맞춤 제안서** 클릭 → 자동 생성된 제안서 확인
5. **③ 안내 문자** 클릭 → 문자 초안 확인

✅ **기대:** 고객 데이터 기반으로 제안서·문자가 자동 생성된다 (LLM 연결 시).

---

## 시나리오 C — 캡처(이미지) OCR
**상황:** 보험 포털/엑셀 화면을 다운로드 못 함 → 화면 캡처.

1. 화면에서 `Win + Shift + S` → 영역 캡처 → 저장(또는 붙여넣기 후 파일로)
2. **💬 AI 질문** 탭 → 그 이미지 파일 추가
3. 질문: `캡처에 있는 내용 알려줘`

✅ **기대:** 이미지에서 글자 추출(OCR). ⚠️ 표·작은 글씨·한글은 부정확할 수 있음(한계 확인 목적).

---

## 시나리오 D — URL에서 가져오기
1. **💬 AI 질문** 탭 → URL 입력칸에 공개 웹페이지 주소 → **가져오기**
2. "✅ 가져옴" 확인 → 그 내용 관련 질문

✅ **기대:** 웹페이지 텍스트를 수집해 질문 가능.

---

## 시나리오 E — 산업 전환 (범용성)
1. **⚙️ 설정** → 산업 카테고리를 **기술**로 변경
2. 대시보드·용어가 "요청자/해결책/처리"로 바뀌는지 확인
3. 다시 **금융**으로 복귀

✅ **기대:** 같은 도구가 산업만 바꿔 다르게 동작.

## VOC 체크리스트 (테스터에게 물어볼 것)
- [ ] 어떤 자료를 제일 많이 넣었나? (엑셀 / PDF / 캡처 / 메모)
- [ ] 어디서 막혔나? (설치 / 자료추가 / 질문)
- [ ] AI 답변이 실제로 쓸 만했나? (정확도 1~5)
- [ ] 캡처(OCR) 정확도는? 녹음은 어떻게 처리하나?
- [ ] 시간 절약됐나? 다시 쓸 건가? (1~5)
- [ ] 빠진 기능 / 가장 원하는 기능?
