# 로그인 후 500 에러 — 세션 캐시 초기화

## 적용 대상 (증상)
- 로그인 직후 대시보드 진입 시 간헐적 500
- 프록시/VPN 환경에서 쿠키 도메인 불일치로 세션 충돌

## 절차
1. 서버 세션 저장소(Redis) 키 prefix 확인
2. 클라이언트 쿠키 도메인 설정을 와일드카드(.example.com)로 통일
3. 세션 캐시 flush 후 재배포

## 주의
- flush 시 전체 사용자 재로그인 발생 → 점검 공지 필요
